
# 提示词驱动交互设计原则

## 核心理念
**"告诉AI你想做什么，而不是找按钮点"**

## 设计变化

### ❌ 旧方式（按钮驱动）
- 固定按钮："引入客户"、"账户开立"
- 表单填写：一步一步填字段
- 用户找功能

### ✅ 新方式（提示词驱动）
- 自然语言："帮我引入华夏理财这个客户"
- AI理解意图，主动推进
- AI找用户，预判需求

## 交互模式

### 1. 自由输入模式
```
用户：刚收到华夏理财的邮件，他们有开户意向

AI：✓ 已识别客户「华夏理财」
   正在创建客户档案...
   
   [卡片] 客户信息预览
   [芯片] 公募基金 | 预计规模5亿 | 联系人张总
   
   建议下一步：资质评估
   [快捷回复] 开始评估 | 补充信息 | 稍后处理
```

### 2. 上下文连续对话
```
用户：开始评估

AI：✓ 正在评估华夏理财资质...
   [进度动画]
   
   评估结果：92分（优秀）
   [卡片] 详细维度得分
   
   建议分配给张经理跟进，是否确认？
   [快捷回复] 确认分配 | 换其他人 | 我自己跟进
```

### 3. AI主动推送
```
AI：⚠️ 提醒：华泰资管资料补充截止时间还有2小时
   
   [卡片] 待补充：法人身份证、授权委托书
   
   需要我帮你催办吗？
   [快捷回复] 一键催办 | 查看详情 | 延后提醒
```

## 界面结构

```

│  🤖 Kimi                │  ← AI头像常驻
│  正在监控12位客户...      │
├─────────────────────────┤
│                         │
│  [对话气泡区域]           │  ← 核心交互区
│                         │
│  AI：今天有3件事...       │
│                         │
│  用户：帮我看看华泰资管    │
│                         │
│  AI：[卡片] 华泰当前状态   │
│      资料待补充，是否催办？ │
│      [催办] [稍后]       │  ← 上下文快捷操作
│                         │
├─────────────────────────┤
│ [语音] 输入你想做的... ➤ │  ← 底部输入栏
│ [快捷芯片：今日待办|紧急] │
└─────────────────────────┘
```

## 关键区别

| 维度 | 按钮驱动 | 提示词驱动 |
|------|----------|------------|
| 用户行为 | 找按钮点 | 打字说话 |
| AI角色 | 被动响应 | 主动理解+推进 |
| 交互流程 | 固定步骤 | 灵活对话 |
| 学习成本 | 高（要熟悉界面） | 低（自然对话） |
| 效率 | 中 | 高（一句话完成） |

## 实现要点

1. **意图识别**：NLP理解用户想做什么
2. **上下文保持**：多轮对话记住状态
3. **主动建议**：AI预判下一步
4. **快捷操作**：在对话中嵌入可选操作
5. **自由输入**：不限制用户表达方式
